SOBRE O PROJETO

Este site tem como objetivo apresentar os produtos de uma empresa de lanches, além de apresentar sobre a empresa e seus principais pontos. A ideia inicialmente era também poder armazenar pedidos etc, porém isso pode ser adicionado futuramente.

O público alvo do site são pessoas consumidoras da lanchonete, que podem ter interesse em saber quais produtos estão disponíveis no Menu, como são feitos, preços e até os possíveis métodos de pagamento. Porém, por ser um conteúdo disponibilizado na internet, pode ser usado para divulgação da empresa e atingir novos clientes e consumidores.

As linguagens utilizadas no projeto foram: HTML5, CSS3 com Bootstrap e JavaScript. Além de tecnologias que auxiliam muito no andamento e desenvolvimento do projeto, como O VS code e suas extensões como: Live server e o Live Share, e para desenho e planejamento inicial do projeto foi usado o Figma.

Os requisitos mínimos é possuir um aparelho com acesso à internet e disponibilidade de navegadores recentes, como Edge, Chrome… entre outros.

As dificuldades encontradas, se deram principalmente pelo versionamento do código e de divisão/organização do projeto. Pelo fato de ser um projeto grande, necessitava da ajuda de todos, então foram separadas as tarefas para não pesar para ninguém. Porém acabou sendo um entrave para todos editarem o código, mesmo usando o live share, ainda foi um entrave, mas poderia ser facilmente resolvido com o uso do Github.

Os pontos fracos do projeto seria em sua maioria a responsividade, uma vez que não é possível utilizar o Grid bootstrap foi pensado o uso do Display flex e o flex basis que pode estar auxiliando nesse processo, mas mesmo assim se torna um dos gargalos do projeto, outro seria a estilização do mesmo. Um ponto forte do trabalho é a facilidade de acessibilidade do site, é bem tranquilo o acesso ao site e fica muito evidente todas informações e subpáginas.

As atualizações do site seriam extremamente importantes por ser uma loja de divulgação de produtos. Pois uma vez que está em falta algum produto, ele não pode estar sendo divulgado como à venda ou disponível. Outro ponto que poderia ser motivo de atualização são as promoções e eventos que surgirem, ou a criação de produtos novos e alteração dos já existentes.

O bootstrap foi usado no projeto exclusivamente para desenvolvimento dos formulários, com intuito de estilizar de forma padronizada os mesmos.
